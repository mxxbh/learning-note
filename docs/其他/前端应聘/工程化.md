# 工程化面试题

## webpack 和 vite 有什么区别？

Webpack 和 Vite 都是前端构建工具，但它们的设计理念、构建方式和使用场景不同。

- 核心区别：Webpack 是基于打包的构建工具，它会在构建阶段对项目中的模块进行依赖分析、打包、处理资源和生成最终产物。Vite 是基于原生 ES 模块的开发服务器和按需编译的构建工具，开发时以快速启动和按需加载为核心，生产环境仍然可使用 Rollup 打包。
- 开发体验：Webpack 开发模式下需要先打包后再启动，启动速度和热更新速度受构建过程影响。Vite 开发模式下使用原生 ES 模块，浏览器直接请求模块，启动速度快，热更新更快。
- 构建方式：Webpack 采用“先打包再执行”，整个依赖图需提前解析，适合大型项目和复杂资源处理。Vite 开发环境采用“按需编译”，只编译当前请求的模块；生产环境通常使用 Rollup 进行打包。
- 配置与插件生态：Webpack 配置灵活，可处理各种 loader 和 plugin，适合复杂定制化场景，但配置相对冗长。Vite 配置简单，内置许多常用功能，插件生态基于 Rollup，通常配置更轻量。
- 适用场景：Webpack 适合复杂项目、老项目、需要细粒度构建控制或自定义 loader 的场景。Vite 更适合现代前端开发、组件库、快速迭代和与 ES 模块生态配合的项目。

总结：
- Webpack 更侧重构建时的模块打包与资源处理。
- Vite 更侧重开发体验和快速启动，生产环境打包由 Rollup 完成。
- 如果追求快速开发体验，优先选择 Vite；如果项目构建需求复杂、兼容性要求高，Webpack 依然是可选方案。
